# Pickty 진행 보관 — 2026-03-29

요약본. 루트 **`PROGRESS.md`** 가 최신 동기화 본문이다.

## P0 검증 정합 (프론트·백엔드)

- **전역 예외**: `GlobalExceptionHandler` — `MethodArgumentNotValidException`·`ConstraintViolationException`·`PicktyValidationException` → UTF-8 `text/plain` 400, DTO `message` 한글 노출.
- **티어 결과 PATCH**: `JsonNode` 병합 유지 + `Validator.validateProperty` + `UpdateTierResultMetaRequest` 프로브, 허용 키 `title`·`description`만.
- **템플릿 생성**: `TemplateItemsPayload` / `TemplateItemPayload` — 템플릿 설명 max 10000, 아이템 이름 max 200 등(Zod와 동일).
- **프론트**: `export-modal` 제목·설명 500/10000·피드백, `PICKTY_IMAGE_ACCEPT` 공통, `pickty-image-accept.ts`.

## 템플릿 CRUD·목록

- **API**: `PUT /api/v1/templates/{id}`(작성자만, 본문은 생성과 동일·**전체 치환**), `DELETE /api/v1/templates/{id}`(작성자 또는 ADMIN). 삭제 시 티어 결과·파생 템플릿 있으면 **409** 한글.
- **목록 DTO**: `TemplateSummaryResponse.creatorId`.
- **프론트**: `/templates` 케밥(수정·삭제 권한 분기), `/template/new?editTemplate=`, `updateTemplate`·`deleteTemplate`, 확인 모달 `template-delete-confirm-dialog`.

## UI

- 티어 카드·결과 상세: **MoreVertical**(⋮), 카드 `overflow-hidden` 제거로 드롭다운 클립 방지, `z-[100]`.
- `/templates` 카드: 제목 1줄 말줄임, 아이템 n개·케밥 **한 줄** 푸터, 패딩·썸네일 높이 축소.

## 용어

- 템플릿 수정 API는 **PUT**(리소스 전체 덮어쓰기). 부분만 바꿀 때는 설계에 따라 PATCH를 쓰기도 함.
