## 진행 메모 **(2026-06-01)**

- **스트리머 모드 Phase 1 (MVP) 구현 + 운영 부하 검증 완료**: 백엔드 — Valkey 세션·`hostToken`·Ticket SSE·동적 백오프(HLL 슬라이딩 윈도우 동접)·ETag 304·IP+visitorId 중복차단·Idle Sweeper. 프론트 — 방장 자동 매치 송출(`use-streamer-host-match-sync`)·결승 자동 종료·실시간 투표 게이지. **운영(`api.pickty.app`) k6 부하 테스트**: 1,200 동접 폴링에서 **에러 0%·p95 160ms·CF차단 0**, 동접 1,050 돌파 시 폴링 **3s→6s 백오프 전환 실측**, **304 ~96%**, 중복투표 차단 확인. 상세 `**docs/STREAMER-MODE-LOADTEST.md**`. 미검증: 2차 전환(3,150)·SSE 1초 스로틀·투표 burst·분산 IP. 현실 최대(5천 시청자→~500 참여)는 여유 판정 → 추가 튜닝은 사용자 유입 전까지 **보류(오버엔지니어링)**.

## 진행 메모 **(2026-04-28)**

- **포트폴리오용 데모 로그인**: 운영 DB에 준비한 `demo-user@pickty.app` 계정을 이메일로 찾아 로그인시키는 `**POST /api/v1/auth/demo-login**` 추가. 기존 OAuth 세션과 동일하게 Refresh **HttpOnly** 쿠키 + Access Token 응답을 발급하며, 프론트 `**/demo-login**` 진입 시 토큰을 `auth-store`에 저장 후 이동. 데모 유저는 `role=USER` 유지, 서버가 `**demoAiEnabled**` 플래그와 AI 자동 생성·사용량 조회 API 예외 권한만 부여해 템플릿 삭제 등 `ROLE_ADMIN` 권한은 열지 않음. 데모 계정 이메일은 `pickty.demo-login.user-email` 설정 기본값으로 관리, 비활성화는 `pickty.demo-login.enabled=false`.
