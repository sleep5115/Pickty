## 1. 개요

| 항목 | 내용 |
|------|------|
| 일시 | 2026-06-01 |
| 대상 | **운영** `https://api.pickty.app` (Cloudflare 프록시 뒤, Lightsail 서울 1GB JVM) |
| 도구 | k6 v2.0.0 (단일 PC, 단일 출구 IP) |
| 세션 | `74fa261a-d5ab-4e34-a528-84bbd3f9f5ed` (WORLDCUP, status=PLAYING, 매치 `83 vs 28`) |
| 범위 | **Phase 1 부하 제어 로직 — 1차 전환만** (백오프 3s→6s, 임계 1,050) |

부하 프로파일: 0 → **1,200 VU** (60s ramp) → 90s 유지 → 15s 감속. 각 VU = 고유 `visitorId` 1개(한 시청자 모사), 받은 `X-Next-Poll-Interval`을 다음 요청 `X-Poll-Interval`에 반영(전환 체인), `If-None-Match`로 304 유도.

안전장치: 에러율 10% 초과(10s 평가) 또는 CF 차단 50건 누적 시 k6 `abortOnFail` 자동 중단.

- 2차: `{accepted:false, duplicate:true, votes:{83:1}}` — 카운트 미증가.

`SHA256(IP + visitorId)` 매치 단위 중복 키 정상 작동.

## 3. 한계 및 미검증 항목

1. **응답시간 꼬리 (max 52.3s)** — p95 160ms로 전반은 건강하나, ramp-down 막판(2분42초~) 소수 요청이 크게 지연. graceful-stop 구간과 겹쳐 단정 불가. 2차 테스트 시 꼬리 확대 여부 관찰 필요.
2. **단일 IP 부하** — HLL은 `visitorId` 다양성으로 카운트해 동접 계측은 유효하나, 실제 수천 개 분산 IP의 Tomcat 커넥션 분산 양상은 미반영.
3. **2차 전환 미검증** — 6s→10s (임계 3,150) 및 히스테리시스 하향 복귀(2,850 / 950) 미측정.
4. **SSE 1초 스로틀 미검증** — 방장 SSE 채널의 throttled push(투표 폭주 시 ≤1회/초)는 hostToken 필요해 별도 진행 예정.
5. **Idle Sweeper / 영속화** — 스케줄러(2h idle, 30분 주기)라 부하 테스트 대상 아님. 설정 단축 또는 단위 테스트로 검증 권장.
